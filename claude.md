# claude.md — Kashani Studio

## What this business is

A boutique web design and development agency run by Pedram.
We design and build custom websites for firms, nonprofits, and service-based businesses.
Core stack: React, Vite, Vercel. No templates, no WordPress, no page builders.

## Who we build for

- Law firms and professional services
- Nonprofits and foundations
- Creative studios and collectives
- Trade and service businesses
- Coaches and personal brands
  Ideal client: established, 2–10 years in, embarrassed by their current site, serious about their image.

## Tone and brand style

- Direct and confident, never corporate or stiff
- Founder-led and personal — this is Pedram, not a faceless agency
- Premium but approachable
- Explains things clearly without condescending
- The brand name "That One Programmer" signals personality — lean into it

## Tech stack

- Frontend: React + Vite
- Hosting: Vercel
- CMS: Sanity or Contentful when needed
- Styling: Custom CSS (no Tailwind by default unless scoped project)
- AI-assisted: Claude for copy drafts, component scaffolding, content strategy

## What the website optimizes for

- One goal: get qualified visitors to book a discovery call
- Two clear audience paths: Premium (custom, high-ticket) and Accessible (financed, starter)
- Packages over a service list — 3 cards max
- Pricing shown as "Starting at..." anchors, not fixed rates
- Social proof: case studies with before/after framing

## What NOT to do

- Do not list every tech skill or service — it dilutes the brand
- Do not offer mobile apps, SaaS, DevOps, or backend systems as marketed services
- Do not use generic agency language ("end-to-end solutions", "synergy", etc.)
- Do not build without a contract and deposit in place
- Do not over-customize Tier 2 (financing) builds — margin lives in speed
- Do not add Phase 2 services before having case studies and client confidence

## Planning before coding

- Always start with a scope doc before writing a line of code
- Wireframe the layout and get approval before moving to build phase
- Define content structure (pages, sections, components) before design decisions
- For CMS projects: design the content schema before the UI

## Testing before deployment

- Test on mobile first, then tablet, then desktop
- Run Lighthouse audit — target 90+ performance, accessibility, SEO scores
- Check all forms, links, and interactive elements manually
- Confirm Vercel deployment preview before pointing live domain
- SEO checklist: meta titles, descriptions, OG tags, sitemap, robots.txt
